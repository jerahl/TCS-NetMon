# Spec 04 — UI port (Phase 4)

**Goal:** the first real dashboards on NetMon's own endpoints — Global,
Switches, and device (AP/switch) Detail — built with an esbuild pipeline from
React source, served as static files by FastAPI. No Zabbix-module coupling, no
CDN / external fetches at runtime (CLAUDE.md §3, Phase 4 DoD).

## Pipeline

- `frontend/` — React source (`src/`), `esbuild` build (`build.mjs`),
  `package.json`. React/ReactDOM are npm deps **bundled locally** by esbuild;
  nothing is fetched from a CDN at build or run time.
- **One command:** `npm --prefix frontend run build` → emits
  `netmon/web/{app.js,app.css,index.html}`.
- The built bundle in `netmon/web/` **is committed** so the deploy needs no
  Node on the VM. `scripts/deploy.sh` rebuilds it when `npm` is present,
  otherwise serves the committed artifact. `frontend/node_modules/` is
  gitignored.

## Serving

FastAPI mounts `netmon/web/` at `/ui` (`StaticFiles`, `html=True`), guarded on
the bundle existing so the API still boots without a build. The SPA is
**hash-routed** (`#/`, `#/switches`, `#/ap/:id`), so one `index.html` serves
every route — deep links never 404 and there is no server-side per-page
routing. Data comes from same-origin JSON (`/api/status`, `/api/devices`); the
session cookie rides along. A 401 renders a "sign in" prompt.

## Pages (this phase)

- **Global** (`#/`) — fleet roll-up tiles (up / down / unknown / source-blind)
  from `/api/status`, plus a problem list of not-fully-healthy devices.
- **Switches** (`#/switches`) — every `device_type=switch` with live
  ping / snmp / XIQ `source_status`.
- **Detail** (`#/ap/:id`) — one device: registry fields, source keys, and the
  three state badges. Per-device XIQ live detail (ports / PoE / clients) is a
  later passthrough endpoint (see below); not stored (§2).

`/api/status` was extended to include the `source_status` dimension so the UI
shows XIQ-federated state alongside the poller's ping/snmp.

## Nav

`src/nav.jsx` ports the reference `global-nav` idiom (collapsible sidebar,
sections, live counts, active highlight) but rewired to NetMon hash routes and
stripped of Zabbix URLs / branding. It carries routes for **all planned pages**
(XIQ, Wireless, Surveillance, VoIP, NAC, Events, Problems, Site Map) even
though only three are built, per the DoD.
